# Пошаговый алгоритм, создания Пользователя на стороне бэк сервиса
1. Получение запроса  
Бэкенд получает POST запрос на /api/users
2. Валидация полей  
Проверить что все обязательные поля заполнены и соответствуют ограничениям. Если нет, то вернуть 422
3. Проверка reCAPTCHA  
Отправить recaptchaToken в Google для проверки. Если невалидный, то вернуть 422
4. Проверка уникальности username  
Проверить в БД существует ли пользователь с таким username. Если существует, то вернуть 409
5. Хэширование пароля  
Захэшировать пароль перед сохранением. Никогда не хранить пароль в открытом виде.
6. Создание пользователя в БД  
Сохранить нового пользователя в БД. Если ошибка БД, то вернуть 500
7. Успешный ответ  
Вернуть 201 Created с данными созданного пользователя
